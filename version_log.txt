[+] Добавлен класс кэширования cache.class.php у которого 4 метода:
    get - возвращает кэшированные данные,
    set - записывает данные в кэш,
    remove - удаляет данные из кэша,
    clear - очищает кэш компонента, раздела компонента или весь кэш;
[+] В Каталог Статей добавлен выбор режима генерации чпу для статей:
        от корня - чпу будет генерироваться от корня раздела без чпу категорий,
        от главной категории - будет использован чпу категории являющейся первым
        (главным) из цепочки категорий материала,
        обычный - чпу будет генерироваться с использованием чпу всей цепочки категорий;
[=] Исправлен метод ядра menuId(). Модуль поставленный для показа например на 
    раздел /new и все его подразделы так же показывался и для разделов /news, 
    /new-story и др.;
[=] В методе ядра detectURI() немного изменено правило игнорирования url начинающихся
    с admin, install, migrate, index, что позволило создавать разделы типа
    /administraciya и они не будут открываться как главная страница;
[+] В настройки системы добавлена вкладка "Кэширование", где можко включить
    кэширование и выбрать тип кэша. Так же настройки объединения css и js файлов
    из вкладки "Дизайн" перенесены сюда же.
[=] Изменены правила объединения css и js файлов. Теперь объединению подвергаются
    только файлы из списка, а не как раньше все js и css файлы (что приводило
    к постоянной перегенерации объединенного файла при появлении новых файлов).
    Перегенерация объединенного файла теперь происходит только при изменении
    какого нибудь из файлов включенных в него.
    Проверка на то изменились ли файлы производится каждый час.
[+] У модулей появилась возможность использовать кеширование внутри модуля, взамен
    системному в классе cmsPage. Это позволяет избежать ошибок связанных с
    с вызовом методов для подключения скриптов и стилей, при кешировании эти
    вызовы просто не проходят.
    Для того чтобы дать системе понять что модуль будет использовать кеширование
    сам, он должен в методе info_module_название_модуля передавать опцию cache_enable=1.
    Для примера можно изучить код модуля "Новые статьи";
[+] В админке добавлен раздел "Настройки шаблонов" где выводится список шаблонов
    с просмотром позиций модулей и настройкой шаблонов без смены шаблона сайта;
[-] В настройках сайта в админке удалена вкладка "Дизайн" ее содержимое перенесено
    во вкладку "Сайт";
[=] При удалении модуля не удалялись позиции модуля из таблицы cms_modules_bind;
[+] В класс rudi_form_generate добавлена поддержка передачи названий и описаний
    полей в массиве $_LANG а не только опциями title и description. Теперь поиск
    названия и описания идет так:
    1. При наличии возвращаются значения опций title и description;
    2. При наличии возвращаются значения $_LANG[TYPE_NAME] и $_LANG[TYPE_NAME_DESC];
    3. Для названия возвращается NAME.
    Для опций полей select и radio для ключа используются еще и префиксы option,
    optgroup и номер опции по счету начиная с нуля:
    $_LANG[SELECT_OPTGROUP_NAME_0]...$_LANG[SELECT_OPTGROUP_NAME_3]...$_LANG[SELECT_OPTGROUP_NAME_N]
    $_LANG[SELECT_OPTION_NAME_0]...$_LANG[SELECT_OPTION_NAME_3]...$_LANG[SELECT_OPTION_NAME_N]
    $_LANG[RADIO_OPTION_NAME_0]...$_LANG[RADIO_OPTION_NAME_3]...$_LANG[RADIO_OPTION_NAME_N]
